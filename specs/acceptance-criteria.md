# Acceptance Criteria（验收标准）

> 每条标准应可测试、可判定通过/失败。
> 使用 Given-When-Then 格式描述行为，粒度控制在单个功能点。

---

## 功能：（功能名称）

> **示例**：批量文件上传

- [ ] **AC-1**：给定用户已登录且有上传权限，当用户上传 1-50 个文件（总大小 ≤ 500MB），那么上传成功并返回文件列表
  - **关联**：PRD「必须做」第 1 条
  - **测试**：`tests/test_upload_small_batch.py`

- [ ] **AC-2**：给定用户正在上传大批量文件（>100 个），当任意文件上传超时（>60s），那么已完成的文件保留，超时文件标记为 failed 并可重试
  - **关联**：PRD「必须做」第 2 条
  - **测试**：`tests/test_upload_timeout.py`

## 边界与错误

- [ ] **AC-E1**：给定用户上传非法文件类型（.exe, .bat），当提交上传请求时，那么返回 400 错误和明确的错误消息
  - **关联**：PRD「安全」要求
  - **测试**：`tests/test_upload_invalid_type.py`

- [ ] **AC-E2**：给定存储服务不可用，当用户发起上传请求时，那么返回 503 错误且不影响已上传文件
  - **关联**：PRD「非功能-兼容性」
  - **降级**：前端显示「服务暂时不可用，请稍后重试」

## 非功能验收

| 指标 | 验收条件 | 测量方式 |
|------|---------|---------|
| 性能 | 100 个文件上传 ≤ 5 分钟 | 压测脚本 + APM 监控 |
| 可靠性 | 上传失败率 < 0.1% | 生产日志统计 |
| 安全 | 无 CVE-high 依赖漏洞 | `pip audit` / `npm audit` |

## 补充说明

- 每条 AC 对应一个或多个测试用例
- 标记 `[ ]` 表示待完成，测试通过后改为 `[x]`
- 无法自动测试的手动验证项，在备注中说明验证步骤
